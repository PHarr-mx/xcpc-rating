# 队伍模块设计

> 关联：[需求文档](../需求文档.txt) · [选手模块](./01-选手模块.md) · [比赛与记录](./03-比赛与记录模块.md)

---

## 1. 模块职责

- 定义**参赛单元**（单人 / 双人 / 三人），人数 **1～3**
- 以**队员集合**识别是否为同一支队伍；**队名可重复**、仅作展示与检索
- 关联正式比赛、训练赛中的排名与获奖；支持正式赛「仅校内队伍」过滤

---

## 2. 领域概念

### 2.1 队伍身份：队员集合（Member Set）

**核心规则**：

| 规则 | 说明 |
|------|------|
| 同一性 | `members` 中 `player_id` **完全相同**（与顺序无关）→ 同一支队伍 |
| 换员 | 任意一人不同 → **另一支队伍**（新 `team_id`） |
| 队名 | **不参与**身份判定；同名不同人、同人不同名，均允许 |
| 人数 | `1`（单挑）、`2`（双打）、`3`（标准队），均合法 |

```
{张三}           → 队伍 A
{张三, 李四}     → 队伍 B
{张三, 李四, 王五} → 队伍 C

同 {张三, 李四, 王五}，比赛 A 叫「三大队」、比赛 B 叫「无敌队」→ 仍是队伍 C
{张三, 李四, 赵六} → 队伍 D（王五 换成 赵六，成员集合变了）
```

### 2.2 队名（Team Name）

- 仅存在于**单场比赛成绩**中的 `team_name`，以及队伍实体上累积的 `names[]`
- **不唯一**、不强制与 `team_id` 一一对应
- 展示时可取「最近使用的队名」或比赛当场队名

### 2.3 长期实体与赛年

- 同一 `member_set` 可跨多个**赛年 / 赛季**参赛，仍是同一 `team_id`
- **赛年 / 赛季**由比赛 `date` 推导，不写在队伍实体上（见 [05-Rating与榜单模块](./05-Rating与榜单模块.md)）

### 2.4 校内队伍（School Team）

- 队员均为本校在籍选手时 `is_school_team: true`（或按配置推断）
- 正式赛只录入本校队伍明细 + `total_teams`；外校不录明细

## 3. 数据模型

### 3.1 规范键 `member_key`

用于匹配与去重，**排序后**拼接：

```text
member_key = "|".join(sorted(player_ids))
# 例：["p003","p001","p002"] → "p001|p002|p003"
```

`member_key` 在 `teams` 表中**唯一**。

### 3.2 `processed/teams.json`

```json
[
  {
    "id": "t_a1b2c3",
    "member_key": "p20230042|p20230015|p20230101",
    "members": ["p20230042", "p20230015", "p20230101"],
    "size": 3,
    "names": ["三大队", "无敌队", "Team Alpha"],
    "display_name": "三大队",
    "is_school_team": true,
    "first_seen": "2024-09-10",
    "last_seen": "2026-06-29"
  },
  {
    "id": "t_d4e5f6",
    "member_key": "p20230042",
    "members": ["p20230042"],
    "size": 1,
    "names": ["张三单独出征"],
    "display_name": "张三单独出征",
    "is_school_team": true,
    "first_seen": "2026-03-01",
    "last_seen": "2026-03-01"
  }
]
```

### 3.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 系统分配，如 `t_{短哈希}` 或 `t001` |
| `member_key` | string | ✅ | 队员集合规范键，**唯一** |
| `members` | string[] | ✅ | 1～3 个 `player_id`，建议存为导入顺序，匹配用 `member_key` |
| `size` | 1 \| 2 \| 3 | ✅ | `len(members)`，便于筛选 |
| `names` | string[] |  | 历次比赛出现过的队名并集 |
| `display_name` | string |  | 默认展示名，如最近一场的 `team_name` |
| `is_school_team` | boolean | ✅ | 是否本校队伍 |
| `first_seen` | date |  | 首次出现在成绩中的日期 |
| `last_seen` | date |  | 最近一场参赛日期 |

**已移除**：`competition_year`、`rosters`（换员即新队，不需时段表）、作为主标识的 `name`。

### 3.4 `team_id` 生成

首次导入某 `member_key` 时：

1. 查 `teams` 是否已有该 `member_key` → 有则复用 `id`
2. 无则分配新 `id`，写入 `teams.json`

## 4. 队伍在比赛记录中的引用

### 4.1 正式比赛

`processed/standings_formal.json`：

```json
{
  "contest_id": "formal_2026_spring_invitational",
  "team_id": "t001",
  "team_name": "三大队",
  "members": ["p20230042", "p20230015", "p20230101"],
  "size": 3,
  "rank": 3,
  "school_rank": 1,
  "award": "gold",
  "solved": 8,
  "penalty": 1200,
  "is_school_team": true
}
```

| 字段 | 说明 |
|------|------|
| `team_id` | 由 `members` 集合解析得到 |
| `team_name` | **该场**使用的队名，可与其他场不同 |
| `members` | 当场队员 1～3 人，应与 `teams.member_key` 一致 |
| `size` | 当场人数 |

### 4.2 校内训练赛

```json
{
  "contest_id": "training_2026_w05",
  "team_id": "t_d4e5f6",
  "team_name": "张三",
  "members": ["p20230042"],
  "size": 1,
  "rank": 5,
  "solved": 3,
  "penalty": 40
}
```

个人训练赛也可用 `player_id` 单行记录（见 [03-比赛与记录模块](./03-比赛与记录模块.md)）；有 `team_name` 时仍可走队伍模型。

---

## 5. 导入与匹配逻辑

1. 解析成绩行：`team_name`（可选）+ `members`（1～3 人，解析为 `player_id`）
2. 计算 `member_key = sorted(members)` 拼接
3. 在 `teams` 中查 `member_key`：
   - **命中** → 使用该 `team_id`；若 `team_name` 非空且不在 `names` 中则追加
   - **未命中** → 新建队伍记录
4. 写入 `standings_*`，带上 `team_id` 与当场 `team_name`

**匹配不依赖**队名、赛年、比赛日期。

`team_id` 若导入文件已提供，仍须校验与 `member_key` 一致，否则报错。

## 6. 对外导出

### 6.1 队伍详情 `public/teams/{id}.json`（可选，二期）

```json
{
  "id": "t_a1b2c3",
  "members": [ /* 含 name、grade 展开 */ ],
  "size": 3,
  "names": ["三大队", "无敌队"],
  "display_name": "三大队",
  "formal_results": [ /* 每条含 team_name、contest 赛年/赛季 */ ],
  "training_results": [ ]
}
```

### 6.2 正式赛比赛页

- `school_standings[]` 含 `team_id`、`team_name`（当场队名）、`members`、`size`

## 7. 校验规则

| 规则 | 处理 |
|------|------|
| `members` 长度 1～3 | 拒绝 |
| `members` 内无重复 `player_id` | 拒绝 |
| 所有 `player_id` 存在于 players 表 | 拒绝或挂起 |
| 同一 `member_key` 仅一条 `teams` 记录 | 合并 |
| 同一场比赛同一 `player_id` 出现两次 | 拒绝（一人一场只能属一个参赛单元） |
| 正式赛外校队伍明细 | 忽略，仅 `total_teams` |

**已移除**：「同一选手在重叠时段不得属两支校队」——同一选手可同时属于 `{A,B,C}` 与 `{A,B}` 等不同 `member_set`；限制改为**单场不重复**。

---

## 8. 与 Rating 的关系

- 队伍成绩映射到选手时，按 `standings.members` 中的 1～3 人分摊或复制事件（算法见 [05](./05-Rating与榜单模块.md)）
- 单挑 / 双打 / 三人队均产生 `team_id`；个人训练赛也可直接用 `player_id` 而不建队
