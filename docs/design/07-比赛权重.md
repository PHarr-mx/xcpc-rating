# 比赛权重（Rating）

> 关联：[05-Rating与榜单模块](./05-Rating与榜单模块.md) · [03-比赛与记录模块](./03-比赛与记录模块.md)  
> 配置文件：[data/config/contest_weights.yaml](../../data/config/contest_weights.yaml)

---

## 1. 总则

- **基准权重为 100**，表示「标准强度」的一场比赛对 Rating 的贡献
- **允许极少数比赛权重 > 100**（如特别重要的决赛）；需在 raw 或 processed 中显式写 `weight_override`，并在 `meta` 注明原因
- 默认权重由配置文件查表；**不在 raw 里逐场手写**（特殊场次除外）
- Rating 算分时：事件得分 × `weight / 100`

---

## 2. 校内训练赛：按 `division` 查权重

与 `format`（`team_xcpc` / `solo_xcpc` / `oi`）**无关**，只看难度档：

| `division` | 名称 | 默认权重 |
|------------|------|----------|
| `div1+2` | 全体队员组 | **100** |
| `div1` | 老队员组 | **95** |
| `div2` | 新队员组 | **70** |
| `div3` | 未入队新生组 | **60** |

raw 示例（无需写 `weight`，import 时自动填充）：

```json
{
  "format": "team_xcpc",
  "division": "div2",
  "standings": [ ]
}
```

---

## 3. 正式比赛：按 `contest_type` 查权重

正式赛在 raw 根节点标注 **`contest_type`**（比赛类型）：

| `contest_type` | 名称 | 默认权重 |
|----------------|------|----------|
| `icpc_regional` | ICPC 区域赛 | **100** |
| `ccpc_national` | CCPC 国赛 | **100** |
| `icpc_invitational` | ICPC 邀请赛 | **80** |
| `ccpc_invitational` | CCPC 邀请赛 | **80** |
| `icpc_online` | ICPC 网络赛 | **90** |
| `ccpc_online` | CCPC 网络赛 | **90** |
| `icpc_provincial` | ICPC 省赛 | **70** |
| `ccpc_provincial` | CCPC 省赛 | **70** |
| `icpc_school` | ICPC 校赛（个人） | **50** |
| `icpc_team_school` | ICPC 组队校赛 | **60** |
| `astar_national` | 百度之星国赛 | **90** |
| `astar_provincial` | 百度之星省赛 | **70** |
| `lanqiao_national` | 蓝桥杯国赛 | **80** |
| `lanqiao_provincial` | 蓝桥杯省赛 | **70** |
| `gplt_national` | GPLT 国赛 | **80** |
| `raicom_national` | RAICOM 国赛 | **75** |
| `raicom_provincial` | RAICOM 省赛 | **65** |

raw 示例：

```json
{
  "title": "2025 ICPC 合肥区域赛",
  "date": "2025-11-20",
  "contest_type": "icpc_regional",
  "total_teams": 120,
  "standings": [ ]
}
```

## 4. 权重解析流程（import 阶段）

```
读取 raw 比赛文件
    │
    ├─ source = training
    │     → weight = contest_weights.training_divisions[division]
    │
    └─ source = formal
          → weight = contest_weights.formal_types[contest_type].weight
          → 若存在 weight_override → 使用覆盖值（可 > 100）

写入 processed/contests_*.json 的 weight 字段
```

```json
{
  "id": "formal_2025_icpc_hefei_regional",
  "contest_type": "icpc_regional",
  "contest_type_label": "ICPC 区域赛",
  "weight": 100,
  "weight_source": "config"
}
```

`weight_source`：`config` | `override`。

---

## 5. 权重 > 100 的例外

极少数场次可手动覆盖：

```json
{
  "contest_type": "ccpc_national",
  "weight_override": 110,
  "weight_override_reason": "本届国赛计入双积分周期"
}
```

校验建议：

- `weight_override > 100` 时要求填写 `weight_override_reason`
- 全年 `weight_override` 场次建议 ≤ 3（可配置）

---

## 6. OJ 记录

OJ 比赛 / 做题权重**不在本表**，由 [05-Rating与榜单模块](./05-Rating与榜单模块.md) 单独配置（默认可设为 30～50 量级，待定）。

---

## 7. 修改权重

1. 编辑 `data/config/contest_weights.yaml`
2. 重新 `xcpc-data import` + `rating` + `export`
3. `meta.json` 记录 `weight_config_version`

不影响 raw 历史文件；processed 重算即可。
