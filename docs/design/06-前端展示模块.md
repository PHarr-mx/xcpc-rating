# 前端展示模块设计

> 关联：[DESIGN.md](../DESIGN.md) · [Rating与榜单](./05-Rating与榜单模块.md) · [选手](./01-选手模块.md)

---

## 1. 模块职责

- Vue 3 静态站的**页面、路由、组件与数据加载**
- 消费 `data/public/` JSON，无运行时后端
- 支持榜单**双模式**与**多时间维度**切换

---

## 2. 技术栈

| 项 | 选型 |
|----|------|
| 框架 | Vue 3 + `<script setup>` + TypeScript |
| 构建 | Vite |
| 路由 | Vue Router 4（history 模式） |
| UI | Naive UI 或 Element Plus |
| 图表 | ECharts + vue-echarts |
| 数据 | `fetch('/data/...')`，开发时代理到 `data/public/` |

---

## 3. 信息架构

```
首页（Rating 榜）
├── 筛选：模式 × 时间维度
├── 排行榜表格
└── 跳转选手 / 比赛

选手详情
├── 基本信息 + OJ 账号
├── Rating 曲线
├── 参赛记录（正式 / 训练 / OJ Tab）
└── 返回榜单

比赛详情（正式 / 训练）
├── 元信息 + total_teams
├── 成绩表（本校或完整）
└── 队内选手链接

关于 / 数据说明
└── meta.built_at、算法版本、赛季定义说明
```

## 4. 路由

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `HomeView` | 默认榜单 |
| `/players/:id` | `PlayerView` | 选手详情 |
| `/contests/formal/:id` | `FormalContestView` | 正式赛 |
| `/contests/training/:id` | `TrainingContestView` | 训练赛 |
| `/about` | `AboutView` | 数据与赛季说明 |

查询参数（首页，可同步到 URL 便于分享）：

```
/?mode=all&period=season&period_id=2025-春学期
```

## 5. 页面设计

### 5.1 首页 — Rating 榜

**布局**

```
┌─────────────────────────────────────────────────────────┐
│  XCPC Rating                                    [关于]   │
├─────────────────────────────────────────────────────────┤
│  数据更新：2026-06-29   算法：placeholder_v0             │
├─────────────────────────────────────────────────────────┤
│  (●) 全部数据  ( ) 仅正式赛                              │
│  [ 生涯 ] [ 2025赛年 ] [ 2025秋学期 ▼ ]                  │
├─────────────────────────────────────────────────────────┤
│  # │ 姓名   │ handle │ 年级   │ Rating │ 近期 │ …        │
│  1 │ 张三   │ 2023级 │ 2456   │ +32  │ link  │          │
│  2 │ 李四   │ 2023级 │ 2410   │ -5   │ link  │          │
└─────────────────────────────────────────────────────────┘
```

**交互**

- 切换模式/周期：重新 `fetch` 对应 JSON（路径来自 `index.json` 或约定规则）
- 列排序：默认按 `rank`；可选按姓名、handle、年级
- **不展示** `status=left` 的选手；`retired` 显示「退役」标记，数据与 Rating 仍保留
- 点击行 → `/players/:id`
- OJ 列：外链到各平台 profile（URL 模板配置在 `frontend/src/config/oj.ts`）

**组件**

- `RatingToolbar.vue` — 模式 + 周期选择
- `RatingTable.vue` — 表格
- `OJHandleLink.vue` — 平台链接

### 5.2 选手详情

```
┌─────────────────────────────────────────────────────────┐
│  ← 返回    张三  2023级                                  │
├─────────────────────────────────────────────────────────┤
│  OJ：CF zhangsan_cf │ AtCoder … │ 洛谷 …                  │
├─────────────────────────────────────────────────────────┤
│  Rating 走势（ECharts）  [全部数据 ▼] [生涯 ▼]            │
├─────────────────────────────────────────────────────────┤
│  [正式赛] [训练赛] [OJ]                                  │
│  日期       比赛              排名    ΔRating             │
│  2026-04-12 春季校赛          #3      +32                 │
└─────────────────────────────────────────────────────────┘
```

**数据**

- `GET /data/players/{id}.json`
- 图表：`rating_history` 过滤后绘制

### 5.3 比赛详情

**正式赛**

- 展示 `total_teams`、本校 `school_standings`
- 队名 → 若做队伍页可扩展；MVP 展示队员链接

**训练赛**

- 标题区展示 `format_label`、`division_label`（如「组队 XCPC · 新队员组」）
- 排行榜列按赛制切换：XCPC 显示 `solved` / `penalty`；OI 显示 `score`
- 组队赛显示队名 + 队员；个人赛显示选手名

### 5.4 关于页

- 读取 `meta.json`
- 静态文案：赛年（9/1–8/31）、赛季划分、Rating 算法说明（链到 docs）

## 6. 数据加载

### 6.1 `lib/api.ts`

```typescript
const BASE = import.meta.env.BASE_URL + 'data';

export async function fetchIndex() {
  return fetchJson<IndexManifest>(`${BASE}/index.json`);
}

export async function fetchRatingBoard(
  mode: 'formal_only' | 'all',
  periodType: string,
  periodId: string,
) {
  const key = periodType === 'career'
    ? 'career'
    : `${periodType}_${periodId}`;
  return fetchJson<RatingBoard>(`${BASE}/ratings/${mode}/${key}.json`);
}

export async function fetchPlayer(id: string) {
  return fetchJson<PlayerDetail>(`${BASE}/players/${id}.json`);
}
```

### 6.2 Composables

| 文件 | 职责 |
|------|------|
| `useRatingCatalog.ts` | 加载 catalogs + index，生成周期下拉选项 |
| `useRatingBoard.ts` | 当前模式/周期下的榜单数据 |
| `usePlayer.ts` | 选手详情 |

### 6.3 加载状态

- 骨架屏或 NSpin
- 404：选手不存在 → 友好提示
- 网络失败：展示 `meta.built_at`，提示数据可能过期

## 9. 响应式与性能

- 校内选手规模（通常 < 500）：全量榜单 JSON 一次加载即可
- 大于 2000 行：按 [04-数据流水线](./04-数据流水线模块.md) 分页，表格用虚拟滚动
- 图表数据点限制：详情页超过 500 点时抽样或按赛年聚合
