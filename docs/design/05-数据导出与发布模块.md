# 数据导出与发布模块设计

> 关联：[DESIGN.md](../DESIGN.md) · [数据导入与加工](./04-数据导入与加工模块.md) · [Rating计算](./06-Rating计算模块.md) · [榜单](./07-榜单模块.md) · [比赛权重](./07-比赛权重.md)

---

## 1. 模块职责

- 将 `data/processed/` 中的规范化数据**计算 Rating 并导出为** `data/public/` 静态文件
- 提供 CLI：`xcpc-data rating | export | update`
- 保证导出**原子性**：失败不破坏上一轮 `public/`

---

## 3. 处理阶段（rating / export / update）

```
processed/ 各表
        │
        ├──────────────────────────────────────────────┤
        │  rating (算法插件)                            │
        ▼                                              │
 ┌──────────────┐    export    ┌──────────────────────▼──┐
 │ rating 状态   │ ──────────▶ │  public/               │
 │ 快照/历史     │             │  榜单+详情+目录          │
 └──────────────┘             └─────────────────────────┘
```

| 阶段 | 命令 | 输入 | 输出 |
|------|------|------|------|
| rating | `xcpc-data rating` | `rating_events` + 选手 | 内存/临时 rating 状态 |
| export | `xcpc-data export` | 全部 processed + rating | `public/**` |
| update | `xcpc-data update` | 以上全部 | 等价于 import→process→rating→export |

> **import、process** 命令由 [数据导入与加工模块](./04-数据导入与加工模块.md) 负责。

## 4. Rating 计算（扩展点）

### 4.1 设计原则

- 算法与流水线**解耦**，通过 `RatingEngine` 插件实现
- MVP 使用 **placeholder**：按简单规则生成可排序数值，便于联调前后端
- 正式算法确定后，只换 `backend/data/rating/` 下实现，不改 export 结构

详细的 Rating 算法见 [06-Rating计算模块](./06-Rating计算模块.md)，榜单模式（`formal_only` / `all`）、时间维度（生涯/赛年/赛季）见 [07-榜单模块](./07-榜单模块.md)。

### 4.2 引擎接口

```python
class RatingEngine(Protocol):
  name: str
  version: str

  def compute(
    self,
    events: list[RatingEvent],
    players: dict[str, Player],
    *,
    mode: Literal["formal_only", "all"],
    period: PeriodFilter,
  ) -> RatingSnapshot:
    ...
```

## 5. 导出层（public）

### 5.1 `meta.json`

```json
{
  "built_at": "2026-06-29T08:00:00Z",
  "data_version": 12,
  "git_sha": "abc1234",
  "rating_algorithm": "placeholder_v0",
  "counts": {
    "players": 156,
    "teams": 48,
    "formal_contests": 8,
    "training_contests": 42
  },
  "rating_modes": ["formal_only", "all"],
  "period_types": ["career", "competition_year", "season"]
}
```

### 5.2 `catalogs/competition_years.json`

```json
[
  {
    "id": 2025,
    "label": "2025赛年",
    "start": "2025-09-01",
    "end": "2026-08-31"
  }
]
```

### 5.3 `catalogs/seasons.json`

```json
[
  {
    "id": "2025-秋学期",
    "competition_year": 2025,
    "kind": "autumn_semester",
    "label": "2025 秋季学期",
    "start": "2025-09-01",
    "end": "2026-01-31"
  },
  {
    "id": "2025-寒假",
    "competition_year": 2025,
    "kind": "winter_break",
    "label": "2025 寒假",
    "start": "2026-02-01",
    "end": "2026-02-28"
  }
]
```

日期边界在 `config.yaml` 的 `calendar` 段配置，processed 阶段写入 catalogs。

### 5.4 榜单文件命名

```
public/ratings/{mode}/{period_type}_{period_id}.json
```

示例：

- `ratings/formal_only/career.json`
- `ratings/formal_only/year_2025.json`
- `ratings/formal_only/season_2025-春学期.json`
- `ratings/all/career.json`

`mode`：`formal_only` | `all`

### 5.5 `public/index.json`

前端启动时可选读取，避免硬编码路径：

```json
{
  "ratings": {
    "formal_only": {
      "career": "/data/ratings/formal_only/career.json",
      "years": ["/data/ratings/formal_only/year_2025.json"],
      "seasons": ["/data/ratings/formal_only/season_2025-春学期.json"]
    },
    "all": { }
  },
  "default_mode": "all",
  "default_period": "career"
}
```

## 6. 原子导出

```python
# 伪代码
def export_public(out_dir: Path):
    staging = out_dir / ".staging"
    write_all(staging)
    validate_schemas(staging)
    atomic_replace(staging, out_dir)
```

失败时保留上一轮 `public/`，并写 `meta.json` 的 `last_error`（可选）。
