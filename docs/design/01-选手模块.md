# 选手模块设计

> 关联：[需求文档](../需求文档.txt) · [队伍模块](./02-队伍模块.md) · [比赛与记录](./03-比赛与记录模块.md)

---

## 1. 模块职责

- 定义选手的**唯一标识**与**静态属性**（姓名、年级、各 OJ 平台 ID）
- 作为 Rating 计算、榜单、队伍成员引用的**主实体**
- 不负责爬取 OJ；仅消费已导入的原始记录并关联到 `player_id`

---

## 2. 领域概念

### 2.1 选手（Player）

校内 XCPC 体系下的参赛个人。一名选手：

- 在本系统内有**校内 ID**（主键）
- 有**姓名**、**handle**（队内称呼/展示用，与 OJ handle 无关）、**年级**
- 有 **`status`**：现役 / 退役 / 离队（见 2.3）
- 可绑定**多个 OJ 平台账号**（0 个或多个）
- 可出现在多支队伍、多场比赛、多条 OJ 记录中

### 2.2 与 OJ 账号的关系

```
Player (1) ──< (N) OJAccount
```

OJ 账号用于：

- 关联 OJ 比赛 Rating、做题数等记录（见 [03-比赛与记录模块](./03-比赛与记录模块.md)）
- 前端展示「各平台 handle」
- **不**用 OJ handle 作为校内系统主键（平台账号可能变更、一人多号）

### 2.3 选手状态（status）

| `status` | 中文 | 展示 | 数据与参赛 |
|----------|------|------|------------|
| `active` | 现役 | ✅ 正常展示 | 正常计入榜单、可参赛 |
| `retired` | 退役 | ✅ 展示（可加「退役」标记） | 历史数据保留；退役后以**非正式选手**身份参赛时，仍可导入成绩并更新 Rating |
| `left` | 离队 | ❌ **默认过滤，不展示** | `processed` 可保留档案供溯源；**不出现在** `public` 榜单、搜索、详情入口 |

离队：因转专业、退学等原因离开集训队，站点当其人不存在。  
退役：毕业或结束正式队员身份，但校友赛、个人参赛等仍可追踪。

### 2.4 校内 handle 与 OJ handle

| 字段 | 层级 | 说明 |
|------|------|------|
| `handle` | 选手 | 校内简称，如 `zs`；榜单、队内展示用，**可选** |
| `oj_accounts[].handle` | OJ 账号 | 各平台用户名，与选手 `handle` 无关 |

### 3.1 `processed/players.json`（中间表）

```json
[
  {
    "id": "p20230042",
    "name": "张三",
    "handle": "zs",
    "grade": 2023,
    "grade_label": "2023级",
    "status": "active",
    "status_label": "现役",
    "oj_accounts": [
      { "platform": "codeforces", "handle": "zhangsan_cf", "user_id": "12345678" },
      { "platform": "atcoder", "handle": "zhangsan_ac", "user_id": null },
      { "platform": "luogu", "handle": "zhangsan_lg", "user_id": "987654" },
      { "platform": "nowcoder", "handle": "zhangsan_nc", "user_id": null }
    ],
    "aliases": ["张三", "Zhang San"],
    "created_at": "2024-09-01",
    "updated_at": "2026-06-29"
  }
]
```

### 3.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 校内选手 ID，全局唯一，建议 `p{入学年}{序号}` |
| `name` | string | ✅ | 显示姓名 |
| `grade` | integer | ✅ | **入学年份**，如 `2023` 表示 2023 级 |
| `grade_label` | string |  | 展示用，默认 `"{grade}级"` |
| `status` | string | ✅ | `active` \| `retired` \| `left`，默认 `active` |
| `status_label` | string |  | 现役 / 退役 / 离队，导出时附带 |
| `oj_accounts` | array |  | 各平台账号列表，可为空 |
| `oj_accounts[].platform` | string | ✅ | 平台枚举，见 3.3 |
| `oj_accounts[].handle` | string | ✅ | 平台用户名 |
| `oj_accounts[].user_id` | string \| null |  | 平台数字 ID（若有），便于去重 |
| `aliases` | string[] |  | 曾用名、英文名，用于导入时模糊匹配 |
| `created_at` | date |  | 建档日期 |
| `updated_at` | date |  | 最近更新 |

### 3.3 `status` 与导出过滤

- **`processed/players.json`**：保留全部状态（含 `left`），便于历史成绩溯源
- **`public/ratings/*.json`**：仅含 `status ∈ { active, retired }` 的选手行
- **`public/players/{id}.json`**：仅当 `status ≠ left` 时生成；`left` 无公开详情页
- 退役选手新成绩：import 照常写入 `standings` / `rating_events`；export 时仍计入其 Rating

### 3.4 OJ 平台枚举

首期支持（可扩展）：

| `platform` | 说明 |
|------------|------|
| `codeforces` | Codeforces |
| `atcoder` | AtCoder |
| `luogu` | 洛谷 |
| `nowcoder` | 牛客 |

新增平台：在 `backend/data` 增加枚举校验，并在 `data/schemas/player.schema.json` 中更新 `platform` 的 `enum`。

### 3.5 年级（grade）约定

- **存储**：统一为**入学年份**（整数），如 2022 级存 `2022`
- **展示**：前端根据当前日期可推算「大几」（可选功能，非 MVP）
- **赛年/赛季筛选**：按记录发生日期 + 选手 `grade` 做统计，不按「大几」存库

## 4. 选手 ID 生成与合并

### 4.1 新建选手

1. 人工或导入文件提供姓名、年级、OJ 账号
2. 系统分配 `id`（或导入文件指定）
3. 写入 `processed/players.json`

### 4.2 导入时匹配（无爬取场景）

从比赛记录、OJ 导出中见到陌生 handle 时：

```
1. 用 (platform, handle) 在 oj_accounts 中精确匹配
2. 用选手 `handle`、姓名 + 年级在 aliases/name 中匹配
3. 仍未命中 → 「待认领」队列（见 4.3）
```

`status=left` 的选手若出现在新成绩中：可关联历史 `player_id` 并写入 processed，但 **export 仍不展示**。

### 4.3 待认领（可选，二期）

`processed/unmatched_accounts.json`：

```json
[
  {
    "platform": "codeforces",
    "handle": "unknown_cf",
    "source": "oj_contest:cf_round_900",
    "seen_at": "2026-06-01"
  }
]
```

管理员手工映射到已有 `player_id` 或新建选手。

## 6. 校验规则

| 规则 | 处理 |
|------|------|
| 同一 `(platform, handle)` 不得绑定两名选手 | 导入报错，需人工合并 |
| 同一 `(platform, user_id)` 若存在且非空，不得重复 | 同上 |
| `id` 唯一 | 重复则拒绝或覆盖（由导入模式决定） |
| `status` 合法枚举 | 拒绝 |
| `left` 不得出现在 `public` 导出 | export 阶段过滤 |
| `grade` 合理范围 | 如 2000–2035，超出警告 |

## 7. 后端代码结构（`backend/data`）

```
src/xcpc_data/
  models/
    player.py          # Pydantic: Player, OJAccount
  processors/
    players.py         # 读取、校验、合并选手表
  import/
    players_import.py    # 从 roster.json 批量导入选手名册
```

## 8. 前端展示要点

见 [06-前端展示模块](./06-前端展示模块.md)：

- 榜单列：姓名、`handle`、年级、状态（退役标记）、Rating、OJ 链接
- 默认不展示 `left`；可选开关「显示退役」仅影响样式，不显示离队
- 选手详情页：基本信息 + 各平台账号链接 + Rating 曲线 + 参赛记录 Tab
